# RTAB-Map 2D/3D 高密度カラーマッピング

このドキュメントでは、Siriusロボットで **RTAB-Map** を使用して、高品質な 2D/3D カラーマップを作成・維持するシステムについて説明します。

## 概要

`sam3_rtabmap.launch.py` は、SAM3 のセグメンテーションデータと RTAB-Map の空間統合能力を組み合わせ、以下の特徴を持つ地図を生成します。

1.  **セマンティック 2D グリッド**: 床面・障害物の識別に加え、SAM3 の意味情報を 2D 地図上に 125 色のインデックスで保持します。
2.  **持続的カラーリング**: 地図の拡張やループクロージャ修正が発生しても、色が消えたり点滅したりしない「リアルタイム・ブリット（転送）」機能を備えています。
3.  **フル環境マッピング (背景対応)**: 壁などの障害物だけでなく、地面や背景を含む全周囲をカラー化可能です。

## 主要な引数

起動時に以下の引数で動作を切り替えられます。

-   `use_sim_time`: シミュレーション環境では `true`（デフォルト）に設定。
-   `include_background`:
    -   `false` (デフォルト): 障害物（壁や物体）のみをマッピング対象にします。
    -   `true`: 床面や背景を含む点群すべてをマッピングに使用します。**床の色を地図に残したい場合に有効です。**

## アーキテクチャ

システムは 3 つのステージでデータを処理します。

### 1. 空間入力 (RTAB-Map ステージ)
-   **入力**: `/sam3/obstacles` または `/sam3/full_cloud`
-   **役割**: SlamToolbox の位置推定を基に、3D 点群を累積し、ループクロージャ（累積誤差の修正）を管理します。

### 2. インデックスカラーマッピング (Node ステージ)
-   **ノード**: `sam3_indexed_map_node`
-   **入力**: `/rtabmap/grid_map` (構造), `/cloud_map` (色情報)
-   **役割**: 3D の点群色情報を 2D の占有格子地図上に投影し、高密度なカラーグリッドを作成します。

### 3. 可視化 (Visualizer ステージ)
-   **ノード**: `sam3_grid_visualizer`
-   **役割**: インデックス化された 2D 地図を RViz2 用にカラー点群 (`/sam3/colored_map_cloud`) として再構成します。

## 配色仕様 (125色パレット)

地図データは ROS 標準の `int8` 型の制限（-128〜127）を考慮し、以下の 125 色パレットを採用しています。

-   **RGB 5段階グリッド (5x5x5 = 125色)**: 各成分を 0, 64, 128, 192, 255 の 5 段階で組み合わせ、豊かな色表現とデータ安全性を両立しています。
-   **予約インデックス**:
    -   `0`: 未探索 (グレー)
    -   `1`: 障害物 (黒)
    -   `2`: 地面 (白)
    -   `3〜127`: SAM3 セグメンテーション由来のカラー

## パフォーマンス最適化設定

`sam3_rtabmap.launch.py` では、実用的な処理負荷に抑えるため以下の設定が適用されています。

-   `RGBD/AngularUpdate`: `0.05` (5度以上の回転で更新)
-   `RGBD/LinearUpdate`: `0.05` (5cm 以上の移動で更新)
-   `Mem/MaxSize`: `2000` (メモリ内のノード数制限による CPU 負荷抑制)
-   `Grid/CellSize`: `0.05` (地図解像度 5cm)

## 注意事項

-   **色の反映**: `include_background:=true` を使用する場合、地面の点群が多くなるため、RTAB-Map のデータベースサイズが増大しやすくなります。
-   **地図の保存**: カラー地図の保存には専用のスクリプト `rtabmap_save.sh` を使用してください。標準の `map_server` ではカラー情報は保存されません。
