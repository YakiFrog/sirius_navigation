# LlmDynamicGoal の概要

`llm_dynamic_goal.py` は、Sirius の自然言語指示を受けて、`Nav2`・`spin`・`cmd_vel`・顔アプリ・状態通知をまとめて扱う対話型ナビゲーションノードです。

## 何をするノードか

- 音声やテキストで来た日本語指示を解析する
- 指示の種類に応じて、前進・後退・旋回・目的地移動・停止・再開を切り替える
- 状況に応じて、顔アプリへ発話や表情を送る
- 障害物、詰まり、バッテリー、直前動作の失敗理由を説明する
- `LLM` は補助で使い、確定できる指示はローカルルールで処理する

## 全体の流れ

1. 入力された文章を正規化する
2. `local_parser.py` でルールベース解析を試す
3. 曖昧なときだけ LM Studio の LLM verifier に確認する
4. 得られたコマンド列を安全に補正する
5. コマンドを順番に実行する

## 解析の考え方

### ルールベースを優先するもの

- `前に行って`
- `後ろに下がって`
- `右向いて`
- `左向いて`
- `1周して`
- `止まれ`
- `バッテリーはどう？`

こうした定型文は、LLM に頼らずローカルルールで決める。

### LLM に回すもの

- 数値や単位が入った複雑な指示
- 文脈依存で曖昧な指示
- 是正・訂正
- 状況説明や雑談

## 実行されるコマンド

### `forward` / `backward`

- `cmd_vel_teleop` に速度指令を流す
- 必要に応じて `cmd_vel` に直接フォールバックする
- 実際の移動距離は `/odom` または TF で計測する
- 障害物が近いときは停止して発話する

### `turn`

- `nav2_msgs/action/Spin` を使う
- その場旋回は `spin` ではなく `turn` に変換して処理する
- 失敗時は、開始角・終了角・障害物距離・status をログに残す

### `spin`

- `360度` 単位の旋回コマンド
- 内部では `turn` に変換して実行する
- `1周` や `一周` は 360 度として扱う

### `goto`

- 地図座標へ移動する
- `前に行け` とは違い、Nav2 のゴールとして扱う
- 経路計画や迂回を許す移動に向く

### `cancel` / `pause` / `resume`

- `pause` は一時停止
- `cancel` は完全キャンセル
- `resume` は一時停止した waypoint の再開

## 前後移動の考え方

前進・後退は、今は「まっすぐ進むこと」を優先している。

- Nav2 の経路計画で迂回はしない
- 代わりに速度を流しながら距離を測る
- 距離は `/odom` フィードバックで測る
- 近接障害物があるときは即停止する

つまり、前後移動は `goto` ではなく、補助付きの直進コマンドとして扱っている。

## 日本語の解釈ルール

### 例

- `行け` 単独: 前進
- `〇〇に行け`: 目的地移動の候補
- `左向いて`: 左旋回
- `左向きに1周して`: 360 度の旋回
- `右向きに90度`: 右旋回

### ルールの方針

- LLM が変な値を返しても、方向の符号は文面側を優先する
- `1周` は LLM の修正対象にしない
- 目的地が明示される文だけ `goto` に寄せる

## ログに出すもの

- 開始時の位置と向き
- 実行コマンドの種類と値
- 旋回の成功・失敗理由
- 障害物の最短距離
- 前後移動の距離計測ソース
- `cmd_vel_teleop` の購読状況

## 経由地（中間地点）の指定と順次移動

「リビングを経由して、庭に行って」のように、指示文の中に複数のランドマーク名が含まれ、移動指示（「〜を経由」「〜に行って」等）がある場合、システムは中間地点を自動で経由しながら目的地へと向かいます。

### 動作仕様

1. **ランドマークの出現順抽出**:
   指示文を左から解析し、出現順にマッチしたランドマーク（例: `[リビング, 庭]`）を取得します。
2. **アナウンスと移動開始**:
   `[happy]リビングを経由し、庭に向かうのだ！` のようにルート全体を発話し、最初の目標地（リビング）へ向けて `publish_direct_map_goal` で Nav2 移動を開始します。
3. **コマンドキューの自動構築**:
   2番目以降の目標地（例: 庭）は、カスタムの発話パラメータ（`"speak": "次は庭に向かうのだ！"`）を持った `goto` コマンドとして順次 `command_queue` にプッシュされます。
4. **自動的な次のアクション実行**:
   経由地に到達したタイミングで、キューから次の目標地が取り出され、ユーザーフレンドリーな発話（「次は〜に向かうのだ！」）を行った上で次の Nav2 アクションを自律的に開始します。

## 使い分けの目安

- 定型で安全に決まるもの: ルールベース
- 曖昧で判断が揺れるもの: LLM
- 危険な動作や方向の符号: ローカルルールを優先
